# Практическое упражнение: Построй IT-ландшафт e-commerce

> **Тема:** 1.1 — Ландшафт IT-систем
> **Инструменты:** draw.io (app.diagrams.net) или Miro

---

## Задание

Вы — системный аналитик в e-commerce компании **«ShopWave»**. Ниже дан список из 10 систем компании. Ваша задача — построить диаграмму IT-ландшафта.

### Шаг 1. Постройте диаграмму (25 мин)

1. Разместите минимум 6 систем на холсте
2. Соедините их стрелками — покажите, какие данные передаются и в каком направлении
3. Подпишите каждую стрелку (что передаётся + как)
4. Визуально выделите категории систем (SoR / SoE / SoI)
5. Добавьте легенду

### Шаг 2. Сравните с эталоном (15 мин)

Модератор покажет эталонную диаграмму. Сравните со своей и ответьте на вопросы:
- Какие связи вы пропустили?
- Какие направления перепутали?
- Где категория системы определена неверно?

### Шаг 3. Заполните шаблон «Описание системы» (10 мин)

Выберите 3 любые системы из ландшафта и заполните шаблон описания:
- Название системы
- Категория (SoR / SoE / SoI)
- Роль в бизнесе (1-2 предложения)
- Ключевые входящие потоки данных
- Ключевые исходящие потоки данных

### Шаг 4. Мини-защита (опционально, 5 мин)

Желающие показывают диаграмму за 2 минуты и отвечают на вопрос:
«Какую бизнес-задачу решает эта связка систем?»

## Подсказки

### Как определить категорию системы

| Категория | Ключевой вопрос |
|-----------|-----------------|
| **SoR** (Record) | Эта система — единственный источник истины для этих данных? |
| **SoE** (Engagement) | Через эту систему пользователь взаимодействует с компанией? |
| **SoI** (Insight) | Эта система анализирует данные для принятия решений? |

### Как рисовать связи

**Синхронная связь (REST, gRPC):** сплошная линия
```
Витрина ────Заказ (REST/JSON)────→ ERP
```

**Асинхронная связь (Kafka, события):** пунктирная линия
```
ERP - - - -Событие "Заказ оплачен"- - - -→ Email Service
```

**Пакетная связь (SFTP, ETL):** линия с точками
```
ERP ····Ночная выгрузка (CSV)····→ DWH
```

### Типичный путь клиента в e-commerce

```
Пользователь → Витрина → ERP → Платёжный шлюз
                                  ↓
Пользователь ← Email ← ERP ← Статус оплаты
```

### Не забывайте параллельные контуры

- **Маркетинг:** CRM ↔ Email-сервис
- **Аналитика:** ERP → DWH → BI
- **Поддержка:** Helpdesk ↔ CRM ↔ ERP
